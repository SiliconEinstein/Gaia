# CLI Overview

> **Status:** Current canonical (alpha 0 grouped CLI)

Gaia's command line is organized by workflow stage. The old flat verbs
(`gaia compile`, `gaia infer`, `gaia register`, ...) are removed; use grouped
paths such as `gaia build compile`, `gaia run infer`, and
`gaia pkg register`.

Authoring starts with the full native Python DSL: run `gaia sdk`, read the
generated `CHEATSHEET.md`, and write the DSL directly. `gaia author` and
`gaia bayes` are optional structured helpers for a subset of that surface.

Use this page to choose the right command family. Use
[Task Guides](cli-task-guides/create-package.md) when you want a step-by-step
path, and [CLI Reference](../reference/cli/index.md) when you need exact
options and command coverage.

## Command Families

| Family | Commands | Use when |
|---|---|---|
| `gaia sdk` | no subcommands | Generate the SDK reference and cheat sheet before direct DSL authoring |
| `gaia build` | `init`, `compile`, `check` | Create a package, compile Python DSL to IR, and validate artifacts |
| `gaia author` | statement verbs + `list` | Optional helper for checked appends into `authored/`; not the full DSL |
| `gaia bayes` | `model`, `compare`, distribution literals | Optional helper for Bayesian model snippets; direct DSL remains primary |
| `gaia pkg` | `add`, `add-import`, `add-module`, `scaffold`, `register` | Install dependencies, manage package modules, scaffold, and publish |
| `gaia run` | `infer`, `render` | Run local inference and emit presentation outputs |
| `gaia inspect` | `starmap`, `starmap-replay` | Visualize compiled package graphs |
| `gaia inquiry` | `focus`, `context`, `review`, obligations, hypotheses, tactics, `reject` | Manage local semantic review work |
| `gaia search` | `lkm ...` | Retrieve LKM knowledge, reasoning chains, nodes, and paper packages |
| `gaia example` | `galileo`, `mendel` | Print runnable example authoring scripts |
| `gaia skill` | `register`, `list` | Materialize bundled Gaia skills into the current project |
| `gaia trace` | `verify`, `review`, `show` | Verify and review ARM execution traces |
| `gaia review` | no leaf commands in alpha 0 | Reserved placeholder for downstream reviewer tooling |

## Reference Boundary

The user-facing task guides intentionally avoid repeating every flag. The
command surface itself is maintained under [CLI Reference](../reference/cli/index.md):

- one page per top-level command family;
- a coverage table for every public leaf command;
- implementation links generated by `mkdocstrings` where useful.

The implementation details behind compile/check/infer/register live under
[Foundational CLI Docs](../foundations/cli/workflow.md). Those pages explain
artifacts, staleness checks, IR hashes, review manifests, and registry metadata;
they are not the first stop for normal command usage.

## Migration

If you have pre-alpha-0 scripts or packages, see
[Migration to alpha 0](../releases/migration-alpha-0.md). It maps removed flat
verbs to grouped commands and also covers Python import-path changes.
