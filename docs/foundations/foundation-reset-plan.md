## Purpose

This file exists only to explain why the foundations reset started.

It is no longer an active execution plan.

## Historical Summary

The reset was originally driven by repeated confusion about:

- what the current product surface actually was
- which shared contracts were real versus aspirational
- whether CLI, server, review, and graph semantics were being described from current code or future design
- how many legacy YAML-era concepts still leaked into active docs

That planning work has now been superseded by the active foundations tree and documentation policy.
