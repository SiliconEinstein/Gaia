> **Status:** Draft
> **Date:** 2026-03-31

## 概述

M2 定义 LKM 的存储层：LanceDB 表 schema、读写操作、Neo4j/Kuzu 拓扑层、StorageManager facade。M2 是 M5（Integrate）、M6（Curation）、M7（BP）、M8（API）的基础。

## 上游参考

- `docs/foundations/lkm/02-storage.md`：存储模型权威来源（写入协议、表结构、降级策略）
- `Gaia/libs/storage/lance_content_store.py`：LanceDB 表初始化、PyArrow schema 定义、读写模式参考
- `Gaia/libs/storage/manager.py`：StorageManager facade 模式参考（三层 store + 写入协议）
- `Gaia/libs/storage/neo4j_graph_store.py`：Neo4j async driver 用法参考（**注意：旧版逐条写是已知性能瓶颈，M2 必须改用 UNWIND**）
- `Gaia/libs/storage/config.py`：StorageConfig 结构参考

## 架构总览

```
StorageManager（facade）
  ├── LanceContentStore   — 必需，content + 参数 + 快照持久化
  │     LanceDB 10 张表
  ├── GraphStore          — 可选（neo4j | kuzu | none），拓扑遍历加速
  └── VectorStore         — 可选（与 LanceDB 共用连接，独立表），embedding 搜索
```

GraphStore 和 VectorStore 缺失时系统优雅降级（回退到 LanceDB 全表扫描）。ContentStore 始终是 source of truth。

## 二、LanceDB 表（10 张）

所有表通过 `LanceContentStore.initialize()` 在首次启动时创建。

### PyArrow schema 约定

- 所有复杂字段（list、dict、嵌套对象）序列化为 JSON 字符串列（`pa.string()`）
- datetime 字段存为 ISO 8601 字符串
- 可选字段存为 `pa.string()`（空值为 `""`）

### 表定义

## 三、写入协议

对齐 `02-storage.md §写入协议`：

```
1. 写入 local_variable_nodes + local_factor_nodes（ingest_status="preparing"）
2. 写入 global_variable_nodes + global_factor_nodes（integrate，确定性去重）
3. 写入 canonical_bindings
4. 写入 prior_records + factor_param_records
5. 写入 graph_store 拓扑（如已配置）
6. 写入 node_embeddings（如已配置）
7. UPDATE local_variable_nodes SET ingest_status="merged"（原子可见性翻转）
```

失败时数据保持 `preparing`，对读端不可见，可安全重试。

**Global 层幂等性**：步骤 2-6 的写入均为 append-only。若 integrate 中途失败，重试时对 global_variable_nodes / global_factor_nodes 的写入需先查询是否已存在（通过 content_hash 或结构匹配），避免重复创建。canonical_bindings 也需检查 `(local_id, package_id, version)` 唯一性后再写入。

**读端过滤**：所有 local node 查询默认加 `WHERE ingest_status = 'merged'` 过滤。

## 四、LanceContentStore

### 4.1 async 包装模式

LanceDB 是同步库，所有调用必须用 `run_in_executor()` 包装：

```python
import asyncio
from functools import partial

async def _run(self, fn, *args, **kwargs):
    loop = asyncio.get_event_loop()
    return await loop.run_in_executor(None, partial(fn, *args, **kwargs))
```

**禁止**在同一 executor 里并行多个写操作（LanceDB 不支持并发写入同一表）。

### 4.2 批量写入

```python
# 正确：一次 add() 写所有行
table.add([row_to_dict(node) for node in nodes])

# 错误：循环逐条写
for node in nodes:
    table.add([row_to_dict(node)])   # 禁止
```

### 4.3 dedup 查找

```python
# content_hash dedup（O(1)，走索引）
results = table.search().where(
    f"content_hash = '{content_hash}' AND visibility = 'public'"
).limit(1).to_list()

# 禁止全表扫描
all_nodes = table.search().limit(1_000_000).to_list()  # 禁止
```

### 4.4 BM25 全文搜索

`local_variable_nodes` 表 `content` 列建 FTS 索引：

```python
table.create_fts_index("content")

# 搜索
results = table.search(text, query_type="fts").limit(top_k).to_list()
```

FTS 索引在表创建后或内容变更后调用 `create_fts_index()` 重建（LanceDB FTS 当前不支持增量更新）。

### 4.5 向量搜索

```python
results = (
    node_embeddings_table
    .search(query_vector, query_type="vector")
    .limit(k)
    .to_list()
)
```

### 4.6 初始化

```python
async def initialize(self) -> None:
    existing = set(self._db.list_tables().tables)
    for table_name, schema in _TABLE_SCHEMAS.items():
        if table_name not in existing:
            await self._run(self._db.create_table, table_name, schema=schema)
    # 建 scalar indexes（表非空后触发，初始化时可能跳过）
    await self._ensure_indexes()
```

## 五、GraphStore（Neo4j / Kuzu）

### 5.1 图模型（global 层）

```
(GlobalVariableNode {gcn_id, type, visibility})
(GlobalFactorNode   {gfac_id, factor_type, subtype})

(GlobalVariableNode)-[:PREMISE   {factor_id}]->(GlobalFactorNode)
(GlobalFactorNode)  -[:CONCLUSION          ]->(GlobalVariableNode)
```

GraphStore 只存 global 层结构，不存 content。Content 始终从 LanceDB 获取。

### 5.2 UNWIND 批量写入（必须）

```python
# 正确：UNWIND 批量
await session.run(
    """
    UNWIND $nodes AS n
    MERGE (v:GlobalVariableNode {gcn_id: n.gcn_id})
    SET v.type = n.type, v.visibility = n.visibility
    """,
    nodes=[{"gcn_id": n.id, "type": n.type, "visibility": n.visibility}
           for n in global_nodes],
)

# 错误：逐条写（已知性能瓶颈，绝对禁止）
for n in global_nodes:
    await session.run("MERGE (v:GlobalVariableNode ...) ...", gcn_id=n.id)
```

**所有 GraphStore 写操作必须用 `UNWIND`**。旧代码（`neo4j_graph_store.py`）的逐条写模式是已知性能瓶颈，不得复用。

### 5.3 降级

`graph_backend="none"` 时：
- `get_subgraph()` 返回空结果
- `get_neighbors()` 返回空结果
- N 跳子图查询回退到 LanceDB 全表扫描（可接受，性能较差）

## 六、StorageManager

```python
class StorageManager:
    def __init__(self, config: StorageConfig) -> None: ...

    async def initialize(self) -> None:
        """初始化所有 store（LanceDB + GraphStore + VectorStore）。"""

    async def close(self) -> None:
        """关闭 GraphStore driver。LanceDB 无需显式关闭。"""

    # ── Ingest 写入（原子，含写入协议） ──
    async def ingest_local_graph(
        self,
        package_id: str,
        version: str,
        variable_nodes: list[LocalVariableNode],
        factor_nodes: list[LocalFactorNode],
    ) -> None: ...

    async def integrate_global_graph(
        self,
        variable_nodes: list[GlobalVariableNode],
        factor_nodes: list[GlobalFactorNode],
        bindings: list[CanonicalBinding],
        prior_records: list[PriorRecord],
        factor_param_records: list[FactorParamRecord],
    ) -> None: ...

    async def commit_package(self, package_id: str, version: str) -> None:
        """将 local nodes 的 ingest_status 从 preparing 翻转为 merged。"""

    # ── 参数化层 ──
    async def write_param_source(self, source: ParameterizationSource) -> None: ...
    async def write_prior_records(self, records: list[PriorRecord]) -> None: ...
    async def write_factor_param_records(self, records: list[FactorParamRecord]) -> None: ...

    # ── 推理输出 ──
    async def write_belief_snapshot(self, snapshot: BeliefSnapshot) -> None: ...

    # ── Embedding ──
    async def write_embeddings(
        self, local_ids: list[str], embeddings: list[list[float]], model: str
    ) -> None: ...

    # ── 读取（variable） ──
    async def get_local_variable(self, local_id: str) -> LocalVariableNode | None: ...
    async def get_global_variable(self, gcn_id: str) -> GlobalVariableNode | None: ...
    async def find_global_by_content_hash(self, content_hash: str) -> GlobalVariableNode | None: ...
    async def find_canonical_binding(self, local_id: str) -> CanonicalBinding | None: ...
    async def find_bindings_by_global_id(self, global_id: str) -> list[CanonicalBinding]: ...

    # ── 读取（factor） ──
    async def get_global_factor(self, gfac_id: str) -> GlobalFactorNode | None: ...
    async def find_global_factor_exact(
        self,
        premises: list[str],
        conclusion: str,
        factor_type: str,
        subtype: str,
    ) -> GlobalFactorNode | None: ...

    # ── 搜索 ──
    async def search_bm25(self, text: str, top_k: int) -> list[LocalVariableNode]: ...
    async def search_vector(
        self, query_embedding: list[float], top_k: int
    ) -> list[tuple[LocalVariableNode, float]]: ...

    # ── 子图 ──
    async def get_subgraph(self, gcn_id: str, hops: int = 2) -> dict: ...
```

## 七、关键约束汇总

1. **`run_in_executor()`**：所有 LanceDB 调用都必须 wrap（LanceDB 是同步库）
2. **批量写**：`table.add(rows)` 写整批，禁止循环逐条
3. **UNWIND**：Neo4j 所有写操作必须用 `UNWIND` 批量执行
4. **content_hash scalar index**：`global_variable_nodes.content_hash` 和 `local_variable_nodes.content_hash` 必须建 scalar index
5. **三向 index on canonical_bindings**：`local_id`、`global_id`、`binding_type` 都建索引，支持双向查询和按类型过滤
6. **ingest_status**：local nodes 写入时初始为 `preparing`，commit 后翻转为 `merged`
7. **global 层不存 content/steps**：`global_variable_nodes` 无 content 字段，`global_factor_nodes` 无 steps 字段
8. **向量索引延迟建立**：LanceDB 向量索引需要在第一批数据写入后建（不是初始化时）
9. **`local_factor_nodes.background` 列**：必须存在，operator 类型存 `""`，strategy 类型存 JSON list
