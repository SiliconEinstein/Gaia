## 概述

M6 异步批量发现 global FactorGraph 中的语义重复、冲突和结构问题，产出结构化提案报告。**不直接修改 global graph**——所有结构变更通过 curation package 走 registry 审查后才生效。

M6 依赖 M2（存储层）+ M5（integrate 完成后的 global graph）。

## 三个发现任务

### 1. Canonicalization Discovery（语义规范化）

发现 global FactorGraph 中**文本不同但语义等价**的 `visibility="public"` variable nodes。

**匹配策略**（content_hash 已在 integrate 阶段处理，此处处理模糊匹配）：

1. **Embedding 余弦相似度**：
   - 从 `node_embeddings` 表获取 embedding 向量
   - LanceDB vector search: `table.search(query_vector).limit(k)`
   - 阈值：cosine similarity > 0.90 → 语义等价候选
   - Embedding 由异步 embedding 组件生成（`asyncio.gather()` + `Semaphore(8)`）

2. **TF-IDF 回退**：
   - 无 embedding 模型时使用 scikit-learn TF-IDF
   - 计算 cosine similarity，阈值相同

**前置过滤**：
- 仅 `visibility="public"` 的 variable 参与
- 仅相同 `type` 的 variable 才有资格匹配
- 含 `parameters` 的全称 claim 额外比较参数结构（count + types 按序匹配，忽略 name）

**判断 Binding vs Equivalence**（对齐 `05-canonicalization.md §2`）：

发现语义等价后，核心问题是**新的推理链是否提供了独立证据**：

| 前提集合关系 | 判断 | 产出 |
|---|---|---|
| 共享前提（前提集合高度重叠） | 未增加独立证据 | **Binding 提案**：建议合并 variable |
| 不同前提（前提集合独立） | 增加独立证据 | **Equivalence 提案**：建议创建 equivalence factor |

```python
@dataclass
class BindingProposal:
    source_gcn_id: str          # 要合并的 variable
    target_gcn_id: str          # 合并目标
    similarity: float           # 相似度分数
    premise_overlap: float      # 前提重叠度
    evidence: dict              # 匹配证据详情

@dataclass
class EquivalenceProposal:
    gcn_id_a: str
    gcn_id_b: str
    similarity: float
    independent_premises: list[str]  # 独立的前提 gcn_ids
    evidence: dict
```

### 2. Conflict Detection（冲突检测）

发现 global FactorGraph 中的矛盾：

**BP 诊断信号**：
- 振荡（`direction_changes` 高）→ 该 variable 从图的不同部分接收矛盾证据
- 高残差（`max_residual` 大）→ BP 未收敛，可能存在冲突

**结构异常**：
- 不一致的 instantiation 关系（schema 与 ground 矛盾）
- 语义矛盾的 claim 对（embedding 相似但 BP 信念值一高一低）

```python
@dataclass
class ContradictionProposal:
    gcn_id_a: str
    gcn_id_b: str
    signal: str          # "oscillation" | "high_residual" | "semantic_conflict"
    evidence: dict       # 诊断详情
```

### 3. Structural Audit（结构审计）

检查 global FactorGraph 健康状况：

```python
@dataclass
class AuditReport:
    orphan_variables: list[str]        # 无任何 factor 连接的 variable gcn_ids
    dangling_factors: list[str]        # premise/conclusion 指向不存在 variable 的 factor gfac_ids
    unresolved_cross_refs: list[dict]  # integrate 时 pending 的跨包引用
    errors: list[str]
    warnings: list[str]
    info: list[str]
```

## Discovery 流程

```
1. 加载 global_variable_nodes（仅 visibility=public）
2. 加载/计算 embeddings（node_embeddings 表）
3. 对每个 variable，搜索 top-k 最近邻（排除自身和已知 binding）
4. 对 similarity > 0.90 的候选对：
   a. 获取各自关联的 factor 的前提集合
   b. 计算前提重叠度
   c. 高重叠 → BindingProposal，低重叠 → EquivalenceProposal
5. 运行 ConflictDetection（需要最近一次 BP 的诊断信息）
6. 运行 StructuralAudit
7. 汇总为 CurationReport
```

## 触发策略

| 模式 | 说明 |
|------|------|
| 批量 | 累积 N 个包后或间隔 T 时间后批量运行 |
| 手动 | 通过 API `POST /curation/run` 触发 |
| 增量（目标） | 只处理自上次 discovery 以来新增的 variable/factor nodes |

当前实现为全量批处理。增量 discovery 是未来优化方向。

---

## 关键约束

1. **不直接修改 global graph**：只产出提案报告，不执行合并/创建
2. **Embedding 并行但限流**：`asyncio.gather()` + `Semaphore(8)`
3. **仅 public variable 参与**：private variable 不参与 curation
4. **相同 type 才比较**：claim 只和 claim 比，setting 只和 setting 比
5. **Binding vs Equivalence 由前提独立性决定**：不能仅凭相似度判断

## 测试要求

### 单元测试

- `test_embedding_similarity_above_threshold`：相似 variable 被发现
- `test_embedding_similarity_below_threshold`：不相似 variable 不被报告
