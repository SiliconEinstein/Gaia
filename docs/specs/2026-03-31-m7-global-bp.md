## 概述

M7 从参数化层按 resolution_policy 解析参数，在 global FactorGraph 上运行 loopy BP，结果写入 BeliefSnapshot。

M7 依赖 M2（存储层）+ M5（integrate 完成后的 global graph）。调用上游 `gaia.bp.BeliefPropagation`（只读，不修改上游）。

## 执行流程

```
1. 加载 global FactorGraph
2. 解析参数（resolution_policy + prior_cutoff）
3. 构建 BP 运行时图
4. 运行 BP
5. 保存 BeliefSnapshot
```

### Step 1: 加载 global FactorGraph

从 M2 存储读取：
- `global_variable_nodes`：所有 variable（含 private）
- `global_factor_nodes`：所有 factor

### Step 2: 解析参数

按 `resolution_policy` + `prior_cutoff` 从参数记录中选择具体值。

**Resolution Policy**：

| policy | 行为 |
|---|---|
| `latest` | 每个 variable/factor 取 `created_at` 最新的记录（限 `created_at <= prior_cutoff`） |
| `source:<id>` | 每个 variable/factor 使用指定 source_id 的记录 |

**source_class 优先级过滤**：

对同一 variable/factor 的多条记录，优先选择高 source_class：

```
official > heuristic > provisional
```

即：有 official 记录时忽略 heuristic/provisional；无 official 时用 heuristic；以此类推。

**参数组装输出**：

```python
@dataclass
class AssembledParams:
    priors: dict[str, float]                    # gcn_id → prior value
    factor_params: dict[str, list[float]]       # gfac_id → conditional_probabilities
    resolution_policy: str
    prior_cutoff: datetime
```

**完整性检查**（对齐 `06-parameterization.md §完整性检查`）：

- 每个 `type=claim`, `visibility=public` 的 variable 必须有 PriorRecord
- 每个 `factor_type=strategy`, `subtype ∈ {infer, noisy_and}` 的 factor 必须有 FactorParamRecord
- 缺失 → 拒绝运行，返回错误

### Step 3: 构建 BP 运行时图

将 LKM 的 `GlobalVariableNode` + `GlobalFactorNode` + `AssembledParams` 转换为上游 `gaia.bp.FactorGraph`：

```python
# 构建 int 索引映射（BP 引擎内部优化）
gcn_to_int: dict[str, int]  # gcn_id → int index
gfac_to_int: dict[str, int]  # gfac_id → int index

# 构建 gaia.bp.FactorGraph
bp_fg = gaia.bp.FactorGraph()
for gcn_id, prior in assembled.priors.items():
    bp_fg.add_variable(gcn_to_int[gcn_id], prior)
for gfac in global_factors:
    bp_fg.add_factor(...)
```

**Int 索引是实现细节，不是存储模型。** 外部系统使用字符串 ID，BP 引擎内部使用 int 索引以提高效率。

### Step 4: 运行 BP

调用上游 `gaia.bp.BeliefPropagation`：

```python
bp = gaia.bp.BeliefPropagation(
    factor_graph=bp_fg,
    damping=0.5,
    max_iterations=50,
    convergence_threshold=1e-6,
)
result = bp.run_with_diagnostics()
```

**BP 参数**：

| 参数 | 默认值 | 说明 |
|---|---|---|
| `damping` | 0.5 | 消息更新混合因子 |
| `max_iterations` | 50 | 迭代上限 |
| `convergence_threshold` | 1e-6 | 收敛阈值 |

### Step 5: 保存 BeliefSnapshot

```python
snapshot = BeliefSnapshot(
    snapshot_id=new_snapshot_id(),
    timestamp=datetime.now(timezone.utc),
    graph_hash=compute_graph_hash(global_variables, global_factors),
    resolution_policy=assembled.resolution_policy,
    prior_cutoff=assembled.prior_cutoff,
    beliefs={gcn_id: result.beliefs[gcn_to_int[gcn_id]]
             for gcn_id in public_claim_ids},  # 仅 public claims
    converged=result.converged,
    iterations=result.iterations,
    max_residual=result.max_residual,
)
```

**beliefs 只包含 `visibility=public` 且 `type=claim` 的 variable。** Private variable 参与 BP 计算但不暴露在 BeliefSnapshot 中。Setting/question 不参与 BP。

## Factor Potential 映射

BP 引擎为每种 factor subtype 使用不同的 potential 函数（对齐 `potentials.md`）：

| factor_type | subtype | potential 类型 | 参数 |
|---|---|---|---|
| strategy | `infer` | 条件 potential | `conditional_probabilities` (2^k) |
| strategy | `noisy_and` | noisy-AND + leak | `[p]`, leak=CROMWELL_EPS |
| strategy | `deduction`~`case_analysis` | 确定性（从结构导出） | 无外部参数 |
| operator | `instantiation` | 确定性蕴含 | 无参数 |
| operator | `equivalence` | 一致/不一致约束 | CROMWELL_EPS |
| operator | `contradiction` | mutex 约束 | CROMWELL_EPS |
| operator | `implication` | 确定性蕴含 | 无参数 |
| operator | 其他 | 确定性约束 | 无参数 |

## 何时运行

| 触发条件 | 说明 |
|---|---|
| Curation 完成后 | 图中的重复和冲突已处理 |
| 手动触发 | `POST /bp/run` |

**不在每次 integrate 后立即运行**：等 curation 处理完语义重复再跑，避免 double counting。

---

## 关键约束

1. **上游 BP 引擎只读**：不修改 `gaia.bp.*`
2. **source_class 优先级不可逆**：official > heuristic > provisional
3. **参数缺失 = 拒绝运行**：不使用默认值
4. **beliefs 只含 public claims**：private variable 和 setting/question 不在输出中
5. **可复现**：相同 graph_hash + resolution_policy + prior_cutoff → 相同 beliefs
