# Gaia v0.8 Spec: Context Reproducibility, Replay, and Diff

## 1. Release theme

```text
Gaia v0.8 — Reproducible Context Release
```

v0.6 introduces `BeliefContext` and `BeliefState`; v0.8 makes them operationally strong: contexts can be locked, replayed, compared, and used as reproducible scientific information states.

Jaynesian target:

```text
Every posterior is P(Claim | I), and I must be inspectable, replayable, and diffable.
```

## 2. Release goals

v0.8 must provide:

```text
1. Stable ContextLock file.
2. Context replay: rerun inference from a saved context.
3. Context diff: compare two contexts and explain what changed.
4. Prior resolution lock.
5. Dependency context lock.
6. Review manifest lock.
7. BeliefState provenance hardening.
```

## 3. New files

Add output:

```text
.gaia/context.json         # current context
.gaia/context.lock.json    # reproducible lock
.gaia/belief_state.json    # posterior + provenance
```

Add source-controlled optional file:

```text
gaia.context.lock.json
```

This file may be committed by package authors when they want exact reproducibility.

## 4. Data contracts

### 4.1 ContextLock

Add:

```text
gaia/ir/context_lock.py
```

```python
class ContextLock(BaseModel):
    schema_version: Literal["gaia.context_lock.v1"] = "gaia.context_lock.v1"

    context_id: str
    created_at: str
    created_by: str | None = None

    package_name: str
    namespace: str
    package_version: str | None = None

    ir_hash: str
    review_manifest_hash: str
    prior_hash: str | None = None

    dependencies: list[DependencyContextRecord] = Field(default_factory=list)
    inference: InferenceEngineConfigRecord

    active_evidence_ids: list[str] = Field(default_factory=list)
    observed_claim_ids: list[str] = Field(default_factory=list)

    policy: dict[str, Any] = Field(default_factory=dict)
```

`ContextLock` is a stable reproducibility artifact. Unlike `BeliefContext`, it should avoid ephemeral metadata.

### 4.2 ContextDiff

```python
class ContextDiff(BaseModel):
    schema_version: Literal["gaia.context_diff.v1"] = "gaia.context_diff.v1"

    left_context_id: str
    right_context_id: str

    ir_changed: bool
    review_changed: bool
    priors_changed: bool
    dependencies_changed: bool
    evidence_changed: bool
    inference_config_changed: bool

    changed_claims: list[str] = Field(default_factory=list)
    changed_actions: list[str] = Field(default_factory=list)
    changed_evidence: list[str] = Field(default_factory=list)
    changed_dependencies: list[str] = Field(default_factory=list)

    summary: str
```

## 5. CLI commands

### 5.1 gaia context lock

```bash
gaia context lock
```

Writes:

```text
.gaia/context.lock.json
```

Optional flags:

```bash
gaia context lock --output gaia.context.lock.json
gaia context lock --include-beliefs
```

### 5.2 gaia context replay

```bash
gaia context replay .gaia/context.lock.json
```

Behavior:

```text
1. Compile current package.
2. Verify current IR hash equals lock.ir_hash.
3. Verify review manifest hash equals lock.review_manifest_hash.
4. Verify dependency context hashes match.
5. Verify prior hash matches.
6. Rerun inference using locked inference config.
7. Produce new BeliefState.
8. Verify new context_id equals lock.context_id.
```

Mismatch behavior:

```text
hard error by default
```

Optional:

```bash
--allow-drift       # not default
--explain-drift     # show diff and exit
```

### 5.3 gaia context diff

```bash
gaia context diff context_a.json context_b.json
```

Output:

```text
Context A: ctx_sha256:...
Context B: ctx_sha256:...
Changed:
  IR: yes/no
  review: yes/no
  priors: yes/no
  dependencies: yes/no
  evidence: yes/no
  inference config: yes/no

Top changed claims:
  ...

Top changed evidence factors:
  ...
```

### 5.4 gaia context inspect


Output:


## 6. Prior resolution hardening

v0.8 must make prior sources explicit.

Add:

```python
class PriorLockRecord(BaseModel):
    prior_hash: str
    source_id: str | None
    records: list[PriorRecord]
    policy: ResolutionPolicy
```

Required behavior:

```text
If a claim prior changes, prior_hash changes.
If resolution policy changes, prior_hash changes.
If prior source changes, prior_hash changes.
```

## 7. Dependency hardening

For each dependency:

```text
dependency package name
namespace
import name
IR hash
context_id if available
belief_state hash if using flat beliefs
mode: flat_beliefs | joint_graph
```

v0.8 should error if:

```text
context lock expects dependency context_id X but current dependency has Y.
```

Unless:

```bash
--allow-drift
```

## 8. BeliefState provenance hardening

`BeliefState` must include:

```text
context_id
context_hash
belief_state_hash
generated_at
inference method actual
exactness
treewidth
active evidence ids
observed claim ids
review manifest hash
prior hash
dependency contexts
```

Add hash rule:

```text
belief_state_hash = SHA256(canonical JSON excluding generated_at and elapsed_ms)
```

## 10. End-to-end workflow

```bash
gaia compile
gaia review accept --all
gaia infer --method jt
gaia context lock --output gaia.context.lock.json

# Later:
gaia context replay gaia.context.lock.json

# Compare:
gaia infer --method bp
gaia context diff .gaia/context.json previous-context.json
```

## 11. Acceptance checklist

```text
[ ] v0.6 and v0.7 tests pass.
[ ] context lock file is written.
[ ] replay succeeds when package state is unchanged.
[ ] replay errors when IR hash changes.
[ ] replay errors when review manifest hash changes.
[ ] context diff identifies IR/review/prior/dependency/evidence/inference config changes.
[ ] BeliefState has stable belief_state_hash.
```

```text
No causal inference.
No distributed package registry protocol.
```
