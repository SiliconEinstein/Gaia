# Gaia v0.10 Spec: Explain, Sensitivity, and Audit

## 1. Release theme

```text
Gaia v0.10 — Scientific Diagnostics Release
```

v0.10 的目标是让 Gaia 不只是计算 posterior，而是解释 posterior、诊断脆弱性、审计证据结构。

Core shift:

```text
From: What is P(claim | context)?
To:   Why is P(claim | context) this value, how fragile is it, and what might be wrong?
```

## 2. Release goals

v0.10 must deliver:

```text
1. gaia explain with direct and path-level explanations.
2. gaia sensitivity for root prior/evidence/review perturbations.
3. gaia audit for evidence, review, priors, graph structure, and dependencies.
4. Diagnostic records included in BeliefState.
5. Machine-readable explanation output.
```

## 3. Explain command

### 3.1 CLI

```bash
gaia explain <claim-label-or-id>
gaia explain <claim> --json
gaia explain <claim> --depth 2
gaia explain <claim> --refresh
```

Default behavior:

```text
Read .gaia/belief_state.json and .gaia/ir.json.
Do not rerun inference unless --refresh is passed.
```

### 3.2 ExplainRecord

Add:

```text
gaia/ir/explain.py
```

```python
class EvidenceExplanation(BaseModel):
    action_id: str
    label: str | None
    evidence_claim_id: str
    hypothesis_claim_id: str
    p_e_given_h: float | None = None
    p_e_given_not_h: float | None = None
    likelihood_ratio: float | None = None
    bayes_factor: float | None = None
    observed: bool
    review_status: str
    source_id: str | None = None
    independence_group: str | None = None
    active: bool

class PriorExplanation(BaseModel):
    claim_id: str
    prior: float
    source_id: str | None = None
    policy: dict[str, Any] = Field(default_factory=dict)

class ExplanationRecord(BaseModel):
    schema_version: Literal["gaia.explanation.v1"] = "gaia.explanation.v1"
    claim_id: str
    label: str | None
    belief: float
    context_id: str
    method_used: str
    is_exact: bool
    treewidth: int | None = None

    direct_evidence: list[EvidenceExplanation] = Field(default_factory=list)
    priors: list[PriorExplanation] = Field(default_factory=list)
    review_summary: dict[str, int] = Field(default_factory=dict)
    path_explanations: list[dict[str, Any]] = Field(default_factory=list)
    warnings: list[str] = Field(default_factory=list)
```

### 3.3 Explain output requirements

Text output must include:

```text
Claim
Belief
Context id
Inference method and exactness
Direct active evidence factors
Inactive/rejected/unreviewed evidence relevant to claim
Root priors directly associated with claim
Review gate summary
Warnings
```

For `--depth 2`, include short path-level influence:

```text
Claim <- evidence <- observed claim
Claim <- derived claim <- upstream prior
```

No need for exact Shapley values in v0.10.

## 4. Sensitivity command

### 4.1 CLI

```bash
gaia sensitivity <claim-label-or-id>
gaia sensitivity <claim> --json
gaia sensitivity <claim> --prior-delta 0.05
gaia sensitivity <claim> --disable-evidence
gaia sensitivity <claim> --toggle-review
gaia sensitivity <claim> --max-runs 100
```

### 4.2 SensitivityRecord

```python
class SensitivityRun(BaseModel):
    perturbation_type: Literal[
        "prior_delta",
        "disable_evidence",
        "toggle_review",
        "disable_dependency",
        "method_change",
    ]
    target_id: str
    description: str
    baseline_belief: float
    perturbed_belief: float
    delta: float

class SensitivityReport(BaseModel):
    schema_version: Literal["gaia.sensitivity.v1"] = "gaia.sensitivity.v1"
    claim_id: str
    context_id: str
    baseline_belief: float
    runs: list[SensitivityRun]
    top_positive: list[SensitivityRun]
    top_negative: list[SensitivityRun]
    warnings: list[str] = Field(default_factory=list)
```

### 4.3 Required perturbations

Minimum:

```text
1. Root prior +delta / -delta for root claims within depth 2.
2. Disable each active direct evidence factor.
3. Toggle accepted review target to inactive for direct evidence.
4. Compare exact/JT/BP if graph small enough.
```

Output sorted by absolute posterior change.

## 5. Audit command

### 5.1 CLI

```bash
gaia audit
gaia audit evidence
gaia audit priors
gaia audit review
gaia audit graph
gaia audit dependencies
gaia audit --json
```

### 5.2 Audit categories

#### Evidence audit

Checks:

```text
duplicate independence_group
observed evidence missing independence_group
evidence missing source_id
evidence metadata schema missing or invalid
Bayes factor / LR extreme without explanation
evidence observed but observation review unaccepted
evidence factor accepted but evidence claim unobserved
```

#### Prior audit

Checks:

```text
claims with implicit neutral prior
claims with extreme prior 0/1 or near 0/1
priors missing source_id
prior source hash mismatch
root priors most influential in sensitivity report
```

#### Review audit

Checks:

```text
unreviewed active targets
rejected targets still influencing BP
accepted observation missing rationale
accepted infer action missing evidence metadata
```

#### Graph audit

Checks:

```text
orphan claims
claims with no prior and no incoming evidence/support
cycles relevant to loopy BP
large treewidth warning
helper claims leaking into public beliefs
operator conclusions without review status if required
```

#### Dependency audit

Checks:

```text
dependency context missing
dependency belief_state stale
dependency IR hash mismatch
foreign claim mapping missing review
shared independence_group across packages
```

## 10. Non-goals

```text
No full causal explanation.
No exact Shapley attribution.
No natural language LLM explanation generation.
No automatic scientific literature audit.
No optimal experiment design yet.
```

## 11. One-line invariant

```text
v0.10 makes Gaia posterior beliefs explainable, stress-testable, and auditable rather than merely computable.
```
