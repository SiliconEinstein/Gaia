# Gaia v0.7 Spec: Scientific Evidence Model Adapters

## 1. Release theme

```text
Gaia v0.7 — Evidence Model Adapter Release
```

v0.6 稳定了 likelihood evidence contract；v0.7 的目标是让用户不必手写 `P(E|H)` / `P(E|¬H)`，而是可以用常见科学数据模型生成 Gaia evidence factors。

核心原则：

```text
Evidence adapters compute or approximate likelihood evidence.
They compile to the existing v0.6 EvidenceMetadata + InferAction contract.
They do not introduce full probabilistic programming.
```

## 2. Release goals

v0.7 must support four built-in adapters:

```text
1. BayesFactorEvidence
2. BinomialEvidence
3. TwoBinomialEvidence
4. GaussianMeasurementEvidence
```

Each adapter must:

```text
1. Accept scientific data inputs.
2. Produce EvidenceMetadata.
3. Produce InferAction-compatible CPT or likelihood ratio.
4. Preserve calculation explanation.
5. Preserve source/provenance/independence metadata.
6. Be review-gated like other evidence actions.
```

## 3. Non-negotiable design constraints

```text
No adapter may directly overwrite claim posterior.
No adapter may assert evidence observed unless observed=True or observe() is called.
No adapter may hide assumptions.
No adapter may silently double-count evidence.
No adapter may erase original statistics after converting them into CPT form.
```

## 4. Data contracts

### 4.1 EvidenceComputationRecord

Add:

```text
gaia/ir/evidence_model.py
```

```python
class EvidenceComputationRecord(BaseModel):
    schema_version: Literal["gaia.evidence_computation.v1"] = "gaia.evidence_computation.v1"

    adapter_kind: Literal[
        "bayes_factor",
        "binomial",
        "two_binomial",
        "gaussian_measurement",
    ]

    inputs: dict[str, Any]
    assumptions: list[str] = Field(default_factory=list)
    output: dict[str, Any]

    method: str
    approximation: str | None = None
    warnings: list[str] = Field(default_factory=list)
```

Attach to:

```python
EvidenceMetadata.generated_from = "adapter:<adapter_kind>"
EvidenceMetadata.query["computation"] = EvidenceComputationRecord
```

or:

```python
IrStrategy.metadata["evidence_computation"] = EvidenceComputationRecord.model_dump()
```

## 5. Adapter APIs

Add:

```text
gaia/lang/dsl/evidence_models.py
```

### 5.1 bayes_factor_evidence

```python
def bayes_factor_evidence(
    evidence: Claim | str,
    *,
    hypothesis: Claim,
    bf: float,
    observed: bool = False,
    source_id: str | None = None,
    data_id: str | None = None,
    model_id: str | None = None,
    independence_group: str | None = None,
    assumptions: list[str] | None = None,
    rationale: str = "",
    label: str | None = None,
) -> Claim: ...
```

Semantics:

```text
BF is treated as LR(H : ¬H) for a binary hypothesis.
```

Compile to v0.6 `bayes_factor()`.

### 5.2 binomial_evidence

```python
def binomial_evidence(
    evidence: Claim | str,
    *,
    hypothesis: Claim,
    successes: int,
    trials: int,
    p_under_h: float,
    p_under_not_h: float,
    observed: bool = False,
    alternative_label: str | None = None,
    source_id: str | None = None,
    data_id: str | None = None,
    independence_group: str | None = None,
    assumptions: list[str] | None = None,
    rationale: str = "",
    label: str | None = None,
) -> Claim: ...
```

Compute:

```text
P(E | H)  = BinomialPMF(successes, trials, p_under_h)
P(E | ¬H) = BinomialPMF(successes, trials, p_under_not_h)
LR        = P(E|H) / P(E|¬H)
```

For large values, compute in log-space.

Validation:

```text
0 <= successes <= trials
0 < p_under_h < 1
0 < p_under_not_h < 1
trials > 0
```

### 5.3 two_binomial_evidence

```python
def two_binomial_evidence(
    evidence: Claim | str,
    *,
    hypothesis: Claim,
    control_successes: int,
    control_trials: int,
    treatment_successes: int,
    treatment_trials: int,
    null_delta: float = 0.0,
    alternative_delta: float | None = None,
    method: Literal["plug_in", "beta_binomial"] = "plug_in",
    observed: bool = False,
    source_id: str | None = None,
    data_id: str | None = None,
    independence_group: str | None = None,
    assumptions: list[str] | None = None,
    rationale: str = "",
    label: str | None = None,
) -> Claim: ...
```

v0.7 supports two methods:

```text
plug_in:
  estimate p_control and p_treatment under point alternatives.

beta_binomial:
  integrate binomial likelihood under Beta priors.
```

Default may be `plug_in`, but output must warn:

```text
"plug_in method is approximate and parameterization-dependent"
```

### 5.4 gaussian_measurement_evidence

```python
def gaussian_measurement_evidence(
    evidence: Claim | str,
    *,
    hypothesis: Claim,
    observed_value: float,
    sigma: float,
    mean_under_h: float,
    mean_under_not_h: float,
    observed: bool = False,
    unit: str | None = None,
    source_id: str | None = None,
    data_id: str | None = None,
    independence_group: str | None = None,
    assumptions: list[str] | None = None,
    rationale: str = "",
    label: str | None = None,
) -> Claim: ...
```

Compute density ratio:

```text
LR = NormalDensity(observed_value; mean_under_h, sigma) /
     NormalDensity(observed_value; mean_under_not_h, sigma)
```

Important:

```text
GaussianMeasurementEvidence uses density ratio, not point probability.
```

Metadata must preserve:

```text
unit
sigma
observed_value
mean_under_h
mean_under_not_h
```

## 6. Numerical requirements

Add:

```text
gaia/math/probability.py
```

Functions:

```python
log_binomial_pmf(k: int, n: int, p: float) -> float
log_normal_density(x: float, mu: float, sigma: float) -> float
logsumexp(values: Sequence[float]) -> float
safe_lr_from_log(log_p_h: float, log_p_not_h: float) -> float
```

All adapters should compute in log-space when values may underflow.

## 7. Compiler and lowering behavior

Adapters compile exactly like v0.6 evidence:

```text
adapter -> EvidenceMetadata + EvidenceComputationRecord -> InferAction -> Strategy(type="infer") -> conditional factor
```

No special BP factor type is required in v0.7.

## 8. Explain output extension

`gaia explain <claim>` should show adapter details:

```text
Evidence model:
  adapter: gaussian_measurement
  observed_value: 5120
  sigma: 80
  mean_under_h: 5200
  mean_under_not_h: 4800
  LR: ...
  approximation: density-ratio
```

## 9. Audit rules

Add warnings:

```text
BayesFactorEvidence missing source_id
BinomialEvidence with very small P(E|H) and P(E|¬H) may be numerically unstable
TwoBinomialEvidence plug_in method used without explicit alternative_delta
GaussianMeasurementEvidence with sigma <= 0 is error
GaussianMeasurementEvidence with unit missing is warning, not error
```

## 11. End-to-end example

```python
from gaia.lang import Claim
from gaia.lang.dsl.evidence_models import gaussian_measurement_evidence

temperature_high = Claim("Cavity temperature exceeds 5000 K.", prior=0.5)
measurement_reported = Claim("Spectrometer reading was 5120 K.")

gaussian_measurement_evidence(
    measurement_reported,
    hypothesis=temperature_high,
    observed_value=5120,
    sigma=80,
    mean_under_h=5200,
    mean_under_not_h=4800,
    unit="K",
    observed=True,
    source_id="instrument:spectrometer_01",
    data_id="measurement:run_2026_04_23",
    independence_group="run_2026_04_23.spectrometer_01.temperature",
)
```

## 13. Non-goals

```text
No p-value to Bayes factor automatic conversion.
No continuous latent variable graph.
No HMC / MCMC backend.
No full unit algebra.
No causal do-operator.
No automatic scientific model selection.
```

## 14. One-line invariant

```text
v0.7 lets scientific data generate likelihood evidence, while preserving Gaia's v0.6 contract that evidence updates claims only through reviewed, contexted likelihood factors.
```
