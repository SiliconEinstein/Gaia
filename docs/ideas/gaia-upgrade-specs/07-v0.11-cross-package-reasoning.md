# Gaia v0.11 Spec: Cross-Package Reasoning Stabilization

## 1. Release theme

```text
Gaia v0.11 — Cross-Package Reasoning Release
```

v0.11 的目标是让 Gaia packages 可以像科学模块一样组合，而不只是像 Python dependencies 一样导入。跨 package reasoning 必须稳定处理 foreign claims、dependency contexts、belief import、joint graph merge、reviewed bridges 和重复证据。

## 2. Release goals

v0.11 must provide:

```text
1. Stable dependency context contract.
2. Foreign claim identity and ownership rules.
3. Bridge/equivalence/relation review contract.
4. Flat-belief import mode hardening.
5. Joint-graph merge mode hardening.
6. Cross-package duplicate evidence detection.
7. Dependency staleness detection.
```

## 3. Core semantics

### 3.1 Package ownership

```text
Every Claim has exactly one owning package namespace.
Local package may reference foreign claims but does not own their priors/posteriors.
Foreign belief may enter local context through explicit dependency context.
```

### 3.2 Two dependency modes

```text
flat_beliefs:
  upstream package posterior beliefs become local priors for foreign claims.

joint_graph:
  upstream package factor graph is merged into local factor graph.
```

Both modes must be context-indexed.

### 3.3 Bridges must be reviewed

Any local claim relation to a foreign claim must have a review target:

```text
local_claim equivalent_to foreign_claim
local_claim contradicts foreign_claim
local_claim fills parameterized foreign claim
local_claim imports foreign claim as premise
```

Review status gates the bridge. It does not become a probability.

## 4. Data contracts

### 4.1 DependencyContextRecord v2

Extend existing record:

```python
class DependencyContextRecord(BaseModel):
    schema_version: Literal["gaia.dependency_context.v2"] = "gaia.dependency_context.v2"

    import_name: str
    package_name: str
    namespace: str

    mode: Literal["flat_beliefs", "joint_graph"]

    ir_hash: str
    context_id: str | None = None
    belief_state_hash: str | None = None
    graph_hash: str | None = None

    version: str | None = None
    source_uri: str | None = None
    resolved_at: str | None = None
```

### 4.2 ForeignClaimRef

```python
class ForeignClaimRef(BaseModel):
    schema_version: Literal["gaia.foreign_claim_ref.v1"] = "gaia.foreign_claim_ref.v1"
    package_name: str
    namespace: str
    knowledge_id: str
    label: str | None = None
    context_id: str | None = None
```

### 4.3 BridgeRecord

```python
class BridgeRecord(BaseModel):
    schema_version: Literal["gaia.bridge.v1"] = "gaia.bridge.v1"

    bridge_id: str
    local_claim_id: str
    foreign_claim: ForeignClaimRef

    bridge_kind: Literal[
        "equivalence",
        "contradiction",
        "implication",
        "parameter_fill",
        "premise_import",
    ]

    review_target_id: str
    rationale: str | None = None
    metadata: dict[str, Any] = Field(default_factory=dict)
```

## 5. CLI behavior

### 5.1 gaia deps inspect

```bash
gaia deps inspect
```

Output:

```text
dependency package
namespace
version
mode
ir_hash
context_id
belief_state_hash
staleness status
```

### 5.2 gaia deps lock

```bash
gaia deps lock
```

Writes dependency context lock section into:

```text
.gaia/context.lock.json
```

### 5.3 gaia deps diff

```bash
gaia deps diff
```

Shows dependency changes:

```text
package version changed
IR hash changed
context_id changed
belief_state hash changed
mode changed
```

### 5.4 gaia infer depth modes

Stabilize:

```bash
gaia infer --depth 0      # local graph + flat dependency beliefs as priors
gaia infer --depth 1      # merge direct dependency graphs
gaia infer --depth -1     # merge transitive dependency graphs
```

Behavior must be explicitly recorded in `BeliefContext.dependencies`.

## 6. Flat-belief import rules

For dependency mode `flat_beliefs`:

```text
1. Load dependency belief_state.json.
2. Verify belief_state.context_id matches dependency context lock if present.
3. Import foreign claim belief as prior for corresponding foreign node.
4. Imported prior source_id must reference dependency context_id.
5. Local evidence must not overwrite foreign package prior unless bridge/review says so.
```

If dependency belief state missing:

```text
warning or error depending on strict mode
```

Strict mode default for locked contexts:

```text
error
```

## 7. Joint-graph merge rules

For dependency mode `joint_graph`:

```text
1. Load dependency IR/factor graph.
2. Namespace all foreign nodes.
3. Merge graph into local factor graph.
4. Apply bridges as reviewed relation factors.
5. Preserve dependency evidence metadata.
6. Preserve dependency review context.
```

If a foreign graph has active evidence with same `independence_group` as local evidence, audit must warn.

## 8. Bridge review rules

Bridge actions must appear in review manifest.

Statuses:

```text
accepted -> bridge factor active
unreviewed -> bridge factor inactive by default
rejected -> bridge factor inactive
```

No bridge may silently activate without review unless it is generated by a trusted package lock and explicitly marked as pre-reviewed.

## 9. Cross-package audit

Extend `gaia audit dependencies`:

Checks:

```text
dependency context missing
dependency belief_state hash mismatch
dependency IR hash mismatch
foreign claim referenced without dependency context
bridge missing review target
bridge unreviewed but expected active
same independence_group across local and dependency evidence
foreign prior imported from stale belief_state
joint_graph merge contains duplicate node identity
```

## 13. Non-goals

```text
No global package registry protocol.
No distributed trust scoring.
No automatic ontology alignment.
No automatic bridge generation by LLM as trusted fact.
No arbitrary package-cycle resolution.
No full semantic version solver.
```

## 14. One-line invariant

```text
v0.11 makes cross-package inference context-stable: imported beliefs, merged graphs, and bridges are all explicitly versioned, reviewed, and auditable.
```
