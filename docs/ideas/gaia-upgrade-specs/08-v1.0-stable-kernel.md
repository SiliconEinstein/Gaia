# Gaia v1.0 Spec: Stable Propositional Scientific Reasoning Kernel

## 1. Release theme

```text
Gaia v1.0 — Stable Kernel Release
```

v1.0 的目标不是把 Gaia 做成完整科学形式化语言终局版，而是发布一个稳定的核心：

```text
claim-centered
action-backed
review-gated
evidence-aware
context-indexed
Jaynesian propositional scientific reasoning kernel
```

## 2. v1.0 product definition

Gaia v1.0 is:

```text
A stable kernel for evaluating P(Claim | Context) in scientific knowledge packages.
```

It supports:

```text
1. Claims as belief variables.
2. Actions as author-declared reasoning/evidence moves.
3. Review manifests as qualitative activation gates.
4. Likelihood evidence and scientific evidence adapters.
5. Contexted, reproducible BeliefStates.
6. Basic quantities, units, and measurement evidence.
7. Explain, sensitivity, and audit tooling.
8. Cross-package reasoning with dependency contexts.
```

## 3. Stable contracts

v1.0 freezes the following public contracts for the 1.x line.

### 3.1 Gaia Lang runtime

Stable:

```text
Knowledge
Claim
Note
Question
Action
Derive
Observe
Compute
Infer
Equal
Contradict
Exclusive
```

Stable API behavior:

```text
Claim is belief variable.
Note and Question are not belief variables.
Action is not a belief variable.
Infer declares likelihood.
Observe declares evidence observation.
Review gates action activation.
```

### 3.2 Evidence contract

Stable:

```text
EvidenceMetadata v1
EvidenceComputationRecord v1
likelihood()
likelihood_ratio()
bayes_factor()
BayesFactorEvidence
BinomialEvidence
TwoBinomialEvidence
GaussianMeasurementEvidence
```

Stable invariant:

```text
Evidence enters through likelihood factors, not posterior assignment.
```

### 3.3 Context contract

Stable:

```text
BeliefContext v1
BeliefState v1
ContextLock v1
ContextDiff v1
```

Stable invariant:

```text
Every belief output is context-indexed.
```

### 3.4 Quantity contract

Stable:

```text
Quantity
Unit
Uncertainty
MeasurementRecord
```

Stable enough for:

```text
numeric values
basic units
basic dimension validation
Gaussian measurement evidence
claim identity / hash with ground values
```

Not stable for:

```text
full SI algebra
symbolic formula unit checking
vector/tensor quantities
```

### 3.5 Cross-package contract

Stable:

```text
DependencyContextRecord v2
ForeignClaimRef v1
BridgeRecord v1
flat_beliefs mode
joint_graph mode
```

## 6. Backward compatibility policy

For v1.x:

```text
1. Schema version changes require migration.
2. CLI command removal requires deprecation for at least one minor release.
3. DSL helper removal requires deprecation for at least one minor release.
4. IR field removal requires schema migration.
5. BeliefState / ContextLock must remain readable across v1.x unless explicitly versioned.
```

## 7. Migration from v0.x

v1.0 must include:

```bash
gaia migrate v0-to-v1
```

Migration targets:

```text
legacy infer metadata -> EvidenceMetadata v1
legacy beliefs.json -> BeliefState v1 where possible
legacy uncontexted outputs -> context_id generation
legacy parameter metadata -> ground value identity metadata
```

## 12. Explicit non-goals for v1.0

v1.0 does not require:

```text
Standalone parser.
Full predicate logic.
Full dependent type system.
Lean/Coq proof backend.
Full continuous probabilistic programming.
HMC/MCMC inference.
Causal do-calculus.
Automatic ontology alignment.
Global package registry.
LLM-generated trusted priors.
Automatic literature ingestion.
```

## 13. Future v1.x roadmap placeholders

Possible post-v1 features:

```text
v1.1: Better path explanations and causal scaffolding.
v1.2: Optional probabilistic-programming backend.
v1.3: More formal unit/dimension algebra.
v1.4: Registry and package trust model.
v1.5: Ontology/claim alignment tools.
v1.6: Experiment design and value-of-information queries.
```

## 14. One-line invariant

```text
Gaia v1.0 is stable when external authors can build scientific knowledge packages that compute, explain, replay, audit, and compose P(Claim | Context) without relying on unstable internal behavior.
```
