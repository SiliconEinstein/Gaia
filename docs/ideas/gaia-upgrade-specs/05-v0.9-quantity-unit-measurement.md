# Gaia v0.9 Spec: Quantity, Unit, and Measurement Semantics

## 1. Release theme

```text
Gaia v0.9 — Scientific Quantity Release
```

v0.9 的目标是让 Gaia 从“科学句子图”推进到“可验证的科学量图”。本版本引入最小可用的 quantity/unit/uncertainty/measurement contract，但不实现完整 dependent type system 或完整单位代数。

## 2. Release goals

v0.9 must support:

```text
1. Quantity values with units.
2. Basic dimension/unit validation.
3. Uncertainty objects.
4. Structured measurement claims.
5. Gaussian measurement evidence using unit-aware inputs.
6. Stable identity/hash for ground parameter values.
```

## 3. Design principles

```text
Quantity is metadata + validation, not a full theorem prover.
Units must be explicit for scientific measurements.
Measurement is evidence, not hard truth.
Observed value must not overwrite true value.
Uncertainty must compile into likelihood when used as evidence.
```

## 4. Runtime types

Add:

```text
gaia/lang/runtime/quantity.py
```

### 4.1 Unit

```python
class Unit(BaseModel):
    symbol: str
    dimension: str | None = None
    scale_to_si: float | None = None
    offset_to_si: float | None = None
```

Minimum built-ins:

```text
Dimensionless
Length: m, cm, mm, km
Mass: kg, g, mg
Time: s, min, h, day
Temperature: K, C
Pressure: Pa, kPa, mmHg
Probability: 1
```

### 4.2 Quantity

```python
class Quantity(BaseModel):
    value: float
    unit: Unit | str

    def to(self, unit: Unit | str) -> "Quantity": ...
```

Validation:

```text
Cannot convert between different dimensions.
Temperature conversion must handle offset.
Probability quantity must be dimensionless and within [0,1] when used as probability.
```

### 4.3 Uncertainty

```python
class Uncertainty(BaseModel):
    kind: Literal["sigma", "interval", "ci", "credible_interval"]
    value: Quantity | tuple[Quantity, Quantity]
    level: float | None = None
    distribution: str | None = None
```

### 4.4 MeasurementRecord

```python
class MeasurementRecord(BaseModel):
    schema_version: Literal["gaia.measurement.v1"] = "gaia.measurement.v1"

    target: str | None = None
    observed_value: Quantity
    uncertainty: Uncertainty | None = None

    instrument_id: str | None = None
    protocol_id: str | None = None
    data_id: str | None = None
    source_id: str | None = None

    assumptions: list[str] = Field(default_factory=list)
```

### 5.4 gaussian_measurement_evidence upgrade

Extend v0.7 API:

```python
def gaussian_measurement_evidence(
    evidence: Claim | str,
    *,
    hypothesis: Claim,
    observed_value: Quantity,
    sigma: Quantity,
    mean_under_h: Quantity,
    mean_under_not_h: Quantity,
    observed: bool = False,
    ...
) -> Claim: ...
```

Rules:

```text
observed_value, sigma, mean_under_h, mean_under_not_h must share dimension.
They may use different units but are converted to common unit before calculation.
Density ratio must account for unit conversion but LR remains dimensionless.
```

## 6. Claim parameter identity / hash requirements

v0.9 must harden ground claim identity.

Rule:

```text
Every bound parameter value must enter either rendered content or canonical identity metadata.
```

For `Claim` subclasses:

```python
class TemperatureAbove(Claim):
    """Temperature of {system} exceeds {threshold}."""
    system: str
    threshold: Quantity
```

The content hash must include:

```text
system value
threshold.value
threshold.unit
```

If content rendering omits a bound parameter, compiler must include it in IR metadata/hash.

## 8. Audit rules

## 11. End-to-end example

```python
from gaia.lang import Claim
from gaia.lang.dsl.measurement import q, sigma, measurement_claim
from gaia.lang.dsl.evidence_models import gaussian_measurement_evidence

temp_high = Claim("Cavity temperature exceeds 5000 K.", prior=0.5)
temp_high.label = "temp_high"

reading = measurement_claim(
    "Spectrometer reading was 5120 K.",
    observed_value=q(5120, "K"),
    uncertainty=sigma(80, "K"),
    instrument_id="spectrometer_01",
    data_id="run_001",
)
reading.label = "spectrometer_reading"

gaussian_measurement_evidence(
    reading,
    hypothesis=temp_high,
    observed_value=q(5120, "K"),
    sigma=q(80, "K"),
    mean_under_h=q(5200, "K"),
    mean_under_not_h=q(4800, "K"),
    observed=True,
    independence_group="run_001.temperature",
)
```

## 13. Non-goals

```text
No complete SI algebra.
No symbolic dimensional analysis for arbitrary formulas.
No tensor/vector units.
No interval arithmetic.
No automatic uncertainty propagation beyond Gaussian measurement adapter.
No continuous latent variable posterior inference.
```

## 14. One-line invariant

```text
v0.9 makes scientific quantities explicit enough that measurements can become validated likelihood evidence rather than ambiguous text claims.
```
