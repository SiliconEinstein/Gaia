# Publish

Fill narrative content into the skeleton generated by `gaia compile . --github`.

## 1. Overview

**Prerequisite:** Run `gaia compile . --github` first. It generates `.github-output/` with:
- `README.md` — skeleton with placeholders
- `narrative-outline.md` — auto-generated reasoning structure (your writing backbone)
- `docs/` — React Pages template with `public/data/sections/` placeholders
- `wiki/` — structured claim reference (already complete, no writing needed)
- `manifest.json` — checklist of what needs to be filled

Your job: read the outline + DSL source + beliefs, write narrative prose.

## 2. Read Your Inputs

```bash
cat .github-output/manifest.json        # What needs filling
cat .github-output/narrative-outline.md  # Writing backbone (auto-grouped sections)
ls src/<package>/*.py                    # DSL source code
cat .gaia/reviews/*/beliefs.json         # BP inference results
```

The `narrative-outline.md` gives you:
- Sections auto-grouped by reasoning connectivity (high cohesion, low coupling)
- Each section lists its claims with prior → belief, derivation chain, and MI bits
- Follow this structure for the README's "Reasoning Structure" section

## 3. Fill README

The README should have this structure when complete:

```markdown
# Package Title

[badges]

> **Reasoning graph information gain: `X.X bits`**    ← auto-generated
> <sub>definition...</sub>                             ← auto-generated

## Summary

[YOUR WRITING: one paragraph, 3-5 sentences]
- What the source material investigates and why it matters
- Core innovation or methodology
- Key results — name exported conclusions with belief values

## Overview

[auto-generated coarse Mermaid graph]                  ← auto-generated

## Reasoning Structure

[YOUR WRITING: one subsection per narrative-outline group]

### [Group Title from outline]
[4-8 sentences telling the STORY of this group's claims]
- What question does this group address?
- Key premises and their confidence (prior → belief)
- How the reasoning connects them
- What BP revealed — which beliefs shifted most and why
- Embed relevant figures inline: ![Caption](docs/public/assets/path)

### [Next Group Title]
...

## Key Findings

[auto-generated conclusions table]                     ← auto-generated

## Weak Points

[YOUR WRITING: 2-4 structurally vulnerable claims]
For each:
- What the claim is and its belief value
- WHY it's weak (structural reason: long chain, weak premise,
  contradiction draining probability, low MI edge, etc.)

## Evidence Gaps

[YOUR WRITING: 2-3 places where additional evidence would help]
For each:
- What evidence is missing
- Which claims/conclusions it would strengthen
```

### Writing the Summary

**Read:** DSL source + beliefs.json + exported conclusions from manifest

Write ONE paragraph. Lead with the research question, then the method, then results.

**Bad:** "This package contains 42 claims organized into 6 modules with 30 strategies..."
**Good:** "This paper develops a parameter-free framework for predicting superconducting Tc by computing the Coulomb pseudopotential from first principles via vDiagMC. The predicted values for Al (1.1K, belief 0.93), Zn (0.7K, belief 0.93), and Li (<0.001K, belief 0.96) closely match experiment, dramatically improving over the phenomenological approach..."

### Writing Reasoning Structure Sections

**Read:** `narrative-outline.md` — each `##` section becomes a `###` subsection in README.

For each outline section:
1. Read the claims listed in that section
2. Read the corresponding DSL source code for full claim content and reason text
3. Write a prose paragraph connecting the claims into a story

**Rules:**
- Do NOT list claims. Tell the narrative.
- Inline prior → belief for key claims: "the adiabatic approximation (0.95 → 0.90)"
- Inline MI for key edges: "this derivation provides 0.30 bits of information"
- If a claim has `metadata.figure`, embed it: `![Caption](docs/public/assets/path)`
- A reader should understand the argument without seeing the knowledge graph

**Bad:** "This section contains adiabatic_approx (prior 0.95, belief 0.90) and cross_term_suppressed (prior 0.90, belief 0.69). They support downfolded_bse via infer."

**Good:** "The downfolding derivation starts from two premises: the adiabatic separation of electron and phonon energy scales (0.95 → 0.90), and the suppression of Coulomb-phonon cross-channel terms at O(ωc²/ωp²) (0.90 → 0.69). Together they yield the frequency-only downfolded BSE (belief 0.76), providing 0.10 bits of information gain. The moderate belief reflects the cross-term estimate's reliance on a plasmon-pole argument without rigorous diagrammatic backing."

### Writing Weak Points

**Read:** narrative outline (low-belief claims, low-MI edges) + beliefs.json

Identify claims that are structurally vulnerable:
- Low belief despite high prior → pulled down by constraints
- Low MI on the edge → reasoning chain adds little information
- Sits on a long derivation chain → multiplicative belief erosion
- Participates in a contradiction → probability drained

For each, explain the MECHANISM of weakness, not just the number.

### Embedding Figures

Scan `graph.json` nodes for `metadata.figure`. For each figure:
1. Check which claim it belongs to
2. Embed it in the Reasoning Structure subsection where that claim appears
3. Write a caption describing what the figure shows

```markdown
![Tc predictions vs experiment for Al, Zn, Li](docs/public/assets/images/fig3.png)
```

## 4. Fill Pages Section Narratives

For each `docs/public/data/sections/*.md`:
- Match it to the corresponding narrative-outline sections (by claim membership)
- Write 4-8 sentences, same quality as README Reasoning Structure
- These are rendered in the interactive GitHub Pages paper

## 5. Chinese Translations

For each `sections/*.md`, create `sections/*-zh.md`.

- Natural Chinese, not literal translation
- Technical terms keep English on first use: "库仑赝势 (Coulomb pseudopotential, μ*)"
- Match narrative structure of English version

## 6. Quality Checklist

Verify before declaring done:
- [ ] All `exported_conclusions` from manifest mentioned in README Summary
- [ ] All `readme_placeholders` filled — no `<!-- ... -->` comments remain
- [ ] Reasoning Structure has one subsection per narrative-outline group
- [ ] Each subsection tells a story (no claim-listing)
- [ ] Figures from `metadata.figure` embedded in relevant sections
- [ ] Weak Points explain WHY each claim is weak (mechanism, not just belief)
- [ ] Evidence Gaps name specific missing evidence
- [ ] Chinese translations exist for all Pages sections
- [ ] README renders correctly in markdown preview
